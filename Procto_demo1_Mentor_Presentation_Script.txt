2. TECH STACK: HOW WE ACHIEVED IT
--------------------------------------------------------------------------------
"To achieve this, we carefully selected a modern web stack:"
- Next.js (App Router): Gives us fast, server-side rendered pages and seamless 
  routing between the Student and Faculty sides.
- Tailwind CSS: We used Tailwind for rapid, utility-first styling. It allowed 
  us to easily map out custom design tokens (like slate-950 for our deep dark 
  mode backgrounds).
- Framer Motion: For all the micro-interactions, like the staggered fade-ups, 
  the floating particles, and the mobile menu animations.
- Supabase: Handles our complex auth layer, enabling Email/Password, Usernames, 
  and Google/GitHub OAuth out of the box.

--------------------------------------------------------------------------------
3. UI DESIGN PHILOSOPHY & AESTHETICS
--------------------------------------------------------------------------------
"Our design system is built around 3 core pillars:"

1. Dark Mode First & Glassmorphism:
   - We used a 'Slate-950' (near-black) background. Instead of flat, solid 
     colors for UI cards, we heavily utilized Glassmorphism (semi-transparent 
     backgrounds with `backdrop-filter: blur`). 
   - HOW WE ACHIEVED IT: By combining Tailwind's `bg-neutral-900/60` with 
     `backdrop-blur-xl` and subtle borders `border-white/10`. This gives the UI depth.

2. Role-Based Color Theming:
   - To prevent users from getting lost, we built dynamic themes. 
   - Faculty Portal: Violet & Cyan (Professional, authoritative).
   - Student Portal: Emerald & Teal (Performance, focus, action-oriented).
   - This theme switches dynamically even on the Login page based on a toggle state.

3. Custom Animations (The "Wow" Factor):
   - We implemented custom React components like `ParticleField` (using HTML5 
     Canvas) for floating background aesthetics.
   - We used `GlitchText` to create a cyberpunk dual-layer color offset.
   - We also added Aurora backgrounds (slow pulsing radial gradients) behind CTAs.

--------------------------------------------------------------------------------
4. PAGE-BY-PAGE FUNCTIONALITY
--------------------------------------------------------------------------------

A. THE LANDING PAGE (/)
- Functionality: The public face of the app showcasing the 6 advanced neural modules.
- Design Hook: Features an infinite CSS Marquee, animated Scroll Counters 
  (counting up to 10000+ exams), and a live mocked "System Target" card to grab 
  attention immediately. It smoothly directs users to their respective login portals.

B. LOGIN & AUTHENTICATION FLOW
- Functionality: Supports Student and Faculty login. We implemented a custom 
  username registry check (mapping `@username` to an email address) allowing 
  users to login with custom handles instead of just emails.
- How we achieved it: A dynamic React state that checks for the presence of the 
  `@` symbol and intercepts the Supabase auth call. The UI shifts from Emerald 
  to Violet instantly via Tailwind transition classes when the role toggle is flipped.

C. FACULTY PORTAL
- Functionality: The command center. Faculty can create classrooms (generating 
  unique PROCTO-XXXX codes) and create quizzes. Quizzes support both MCQ and 
  Descriptive formats with dedicated forms.
- Design Hook: The dashboard uses a 2-column hero layout with "glass-cards". 
  Hovering over actions (like 'Create Quiz') activates a 3D tilt effect and an 
  animated border glow, giving immediate tactile feedback.
- How we achieved it: Data is dynamically stored and retrieved using browser 
  `localStorage` to simulate backend persistence without lag during this demo phase.

D. STUDENT PORTAL
- Functionality: Highly focused environment. Students can use codes (like QUIZ-XXXX) 
  to bridge into live exams or classrooms.
- Design Hook: Instead of plain backgrounds, the Student Hub incorporates a 
  Carbon Fiber CSS pattern overlay and pulsing "Live Profile" indicators.
- Exam Room: Built with a mock countdown timer (e.g., 30 mins defaults to 1800 
  seconds), auto-submit triggers, and a distraction-free full-screen layout.

--------------------------------------------------------------------------------
5. STATE MANAGEMENT & ARCHITECTURE
--------------------------------------------------------------------------------
"Mentor, you might ask how the data flows right now:"
- For this v3.0 demo, to ensure zero-latency testing, Classroom and Quiz data 
  (like the unique generated codes, questions, and UI state) are mapped directly 
  in the browser's `localStorage` (`procto_quizzes`, `procto_user_registry`, etc.).
- Authentication state is strictly managed by Supabase cookies, and we have a 
  Next.js `middleware.ts` file acting as a bouncer. If a student tries to access 
  the /faculty route, the middleware intercepts the request at the edge and 
  redirects them back to safety.
