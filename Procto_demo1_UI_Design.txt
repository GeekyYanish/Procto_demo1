================================================================================
1. PROJECT OVERVIEW
================================================================================

Procto is a Next.js-based AI-powered exam proctoring web application designed
for Christ University's MCA program. The application provides a futuristic,
cyberpunk-inspired dark-themed interface with distinct portals for Faculty and
Students. The UI prioritizes a premium, high-tech aesthetic using glassmorphism,
particle effects, glitch animations, and smooth micro-interactions.

Tech Stack:
  - Framework      : Next.js (App Router)
  - Language        : TypeScript (TSX)
  - Styling         : Tailwind CSS + Custom CSS
  - Animation       : Framer Motion
  - Authentication  : Supabase (Email/Password + Google OAuth + GitHub OAuth)
  - Icons           : Lucide React (icon library)
  - Fonts           : Inter (UI text) + JetBrains Mono (monospace/code style)


================================================================================
2. DESIGN SYSTEM & GLOBAL STYLES
================================================================================

2.1 COLOR PALETTE
-----------------
The application uses a carefully curated dark-mode palette:

  Primary Background    : #020617 (Slate-950 / near-black)
  Primary Foreground    : #f1f5f9 (Slate-100 / off-white)
  Accent Cyan           : #22d3ee (Cyan-400) — primary accent for CTAs & highlights
  Accent Violet         : #a78bfa (Violet-400) — secondary accent for faculty theme
  Accent Emerald        : #34d399 (Emerald-400) — student portal identity color

  Role-Based Color Themes:
    • Landing Page      : Cyan + Violet gradient on Slate-950
    • Faculty Portal    : Violet + Cyan on Slate-950
    • Student Portal    : Emerald + Teal on Neutral-950
    • Login Page        : Dynamic — switches between Emerald (Student) and
                          Violet (Faculty) based on selected role

2.2 TYPOGRAPHY
--------------
  Primary Font  : 'Inter' (weights: 300–800) — Clean, modern sans-serif
  Mono Font     : 'JetBrains Mono' (weights: 400, 700) — Used for labels,
                  status indicators, version tags, and code-like elements
  Heading Style : Extra-bold (font-black), tight letter-spacing (tracking-tighter)
  Body Style    : Regular weight, relaxed line-height, slate-400 color

2.3 GLASSMORPHISM (Core Visual Pattern)
----------------------------------------
Used extensively across all pages:

  • glass class:
      - Background: rgba(15, 23, 42, 0.6) — semi-transparent dark slate
      - Backdrop-filter: blur(20px) saturate(180%)
      - Border: 1px solid rgba(255, 255, 255, 0.08)
      - Box-shadow: 0 8px 32px rgba(0, 0, 0, 0.37)

  • glass-card class:
      - Background: rgba(15, 23, 42, 0.5) with blur(12px)
      - Hover: border glow to cyan, translateY(-2px) lift effect
      - Smooth cubic-bezier transitions (0.4s)

2.4 CUSTOM ANIMATIONS
-----------------------
  • Aurora Background   : Slow-moving gradient animation (15s cycle)
  • Scanline Overlay    : Retro CRT-style scanning line effect (8s cycle)
  • Glitch Text Effect  : Dual-layer glitch with cyan & violet offset clips
  • Particle Field      : Floating particles across the landing page background
  • Shimmer Button      : Moving light streak across CTAs (3s cycle)
  • Pulse Glow          : Breathing glow effect on cards (3s cycle)
  • Border Glow         : Animated border color pulse (3s cycle)
  • Flicker             : Subtle opacity flicker for status indicators (2s cycle)
  • Float Reverse       : Gentle floating motion for decorative elements (4s)
  • 3D Tilt Hover       : perspective(1000px) rotateX/Y on hover with scale(1.02)
  • Marquee             : Infinite horizontal scroll for keyword ticker (20s)
  • Stagger Container   : Children animate in with 0.1s delay between each
  • FadeUp              : Elements slide up 40px with opacity fade (0.6s)

2.5 SCROLLBAR & SELECTION
--------------------------
  • Custom scrollbar: 4px width, gradient thumb (cyan -> violet)
  • Selection color: cyan background with dark text
  • Smooth scroll behavior enabled globally

================================================================================
4. CUSTOM COMPONENTS
================================================================================

  a) ParticleField (ParticleField.tsx)
     - Canvas-based floating particle animation
     - Renders across the landing page background
     - Creates an ambient, futuristic atmosphere

  b) GlitchText (GlitchText.tsx)
     - Text component with dual-layer glitch animation
     - Cyan and violet color offset layers
     - Uses CSS clip-path polygons for the glitch strips

  c) AnimatedCounter (AnimatedCounter.tsx)
     - Animates numerical values counting up on scroll
     - Used in the stats section of the landing page
     - Supports formats like "99.7%", "50ms", "10000+"

  d) Navbar (Navbar.tsx)
     - Reusable navigation bar component
     - Glass background with responsive mobile menu


================================================================================
5. RESPONSIVE DESIGN STRATEGY
================================================================================

  • Mobile-First approach using Tailwind's responsive prefixes:
    - sm: (640px), md: (768px), lg: (1024px), xl: (1280px)
  • Key Breakpoint Behaviors:
    - Mobile: Single column, hamburger menu, simplified cards
    - Tablet (md): Two-column grids, visible navigation
    - Desktop (lg): Full layout with side-by-side hero sections
    - Large (xl): Max-width containers, increased padding
  • Touch-Friendly: Large tap targets (px-8 py-4 on CTAs), minimum 44px hit areas


================================================================================
6. AUTHENTICATION FLOW (UI Perspective)
================================================================================

  1. User lands on Landing Page (/)
  2. Clicks "Faculty Login" or "Student Login" → Navigates to /login?role=faculty/student
  3. Login page pre-selects the correct role toggle
  4. User can:
     a. Sign in with Email + Password
     b. Sign up with Email + Password
     c. Sign in with Google OAuth
     d. Sign in with GitHub OAuth
  5. On success → Redirects to /faculty or /student based on role
  6. Middleware protects /faculty and /student routes (requires auth)
  7. Profile dropdown on dashboards provides Logout functionality


================================================================================
7. UI DESIGN PRINCIPLES FOLLOWED
================================================================================

  1. DARK MODE FIRST       → Reduces eye strain, premium feel
  2. GLASSMORPHISM          → Depth without visual clutter
  3. ROLE-BASED THEMING     → Distinct visual identity per user type
  4. MICRO-INTERACTIONS     → Hover effects, transitions, animated elements
  5. PROGRESSIVE DISCLOSURE → Dashboard cards reveal details on interaction
  6. CONSISTENT SPACING     → Tailwind's spacing scale for uniform layouts
  7. ACCESSIBILITY          → Semantic HTML, focus states, heading hierarchy
  8. PERFORMANCE            → On-device animations, lazy-loaded components

PRESENTATION TIPS
================================================================================

  1. TOOL RECOMMENDATION:
     - Use Google Slides, Canva, or PowerPoint
     - For a premium look, use Canva's dark-themed templates and customize
       with the Procto color palette (cyan/violet/emerald)

  2. SCREENSHOT CAPTURE:
     - Run the project locally (npm run dev) and take full-page screenshots
     - Use browser DevTools to capture at different viewport widths
     - For animations, record short screen captures (GIFs or MP4s)

  3. CONSISTENCY:
     - Use the same dark background (slate-950 or #020617) across all slides
     - Use Inter font in the presentation for consistency with the project
     - Use the accent colors (cyan, violet, emerald) for highlights

  4. ANNOTATIONS:
     - Add numbered callout circles on screenshots
     - Use thin arrow lines pointing to specific UI elements
     - Keep annotation text brief (2-5 words each)

  5. TIMING:
     - Aim for 15-20 minutes total presentation
     - Spend ~1 minute per slide maximum
     - Allow 3-5 minutes for Q&A at the end

  6. DEMO:
     - If possible, include a live demo of the running application
     - Navigate through the key flows: Landing → Login → Dashboard → Actions
     - Show the animations and hover effects in real-time
